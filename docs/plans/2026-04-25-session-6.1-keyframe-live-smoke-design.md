## 4. Components

### 4a. `RemoteAestheticScorer` (rewritten)

```python
class RemoteAestheticScorer:
    """LAION-Aesthetics v2 scorer. Calls a FastAPI score_server running on the
    vast.ai box (see scripts/score_server/server.py).

    Lifecycle: instantiated once per Stage.run; closed via aclose() at end of stage.
    Mirrors HttpComfyClient -- transport seam for httpx.MockTransport in unit tests.
    """

    def __init__(
        self,
        *,
        host: str,                           # e.g. "http://1.2.3.4:8189"
        timeout: float = 30.0,
        transport: httpx.AsyncBaseTransport | None = None,
    ) -> None:
        if not host:
            raise ValueError(
                "RemoteAestheticScorer requires a host "
                "(set PLATINUM_AESTHETICS_HOST in .env or aesthetics.host in settings.yaml)"
            )
        self._host = host.rstrip("/")
        self._client = httpx.AsyncClient(
            base_url=self._host,
            timeout=timeout,
            transport=transport,             # MockTransport in tests; AsyncHTTPTransport in prod
        )

    async def score(self, image_path: Path) -> float:
        with image_path.open("rb") as fh:
            files = {"image": (image_path.name, fh, "image/png")}
            resp = await self._client.post("/score", files=files)
        resp.raise_for_status()
        body = resp.json()
        score = float(body["score"])
        if not math.isfinite(score):
            raise ValueError(f"score_server returned non-finite score: {body!r}")
        return score

    async def aclose(self) -> None:
        await self._client.aclose()
```

`KeyframeGeneratorStage.run` adds `try/finally: await scorer.aclose()` if
`scorer` was constructed by the Stage (i.e., not injected via
`test_overrides`).

### 4b. `scripts/score_server/server.py`

```python
import io, os
from pathlib import Path
import torch, open_clip
from fastapi import FastAPI, File, UploadFile, HTTPException
from PIL import Image

DEVICE = "cuda" if torch.cuda.is_available() else "cpu"
MLP_WEIGHTS = Path(os.environ.get(
    "LAION_MLP_PATH",
    "/workspace/models/aesthetic/sac+logos+ava1-l14-linearMSE.pth",
))

# --- Model load happens once at import; Uvicorn keeps the process alive --------

clip_model, _, preprocess = open_clip.create_model_and_transforms(
    "ViT-L-14", pretrained="openai", device=DEVICE
)
clip_model.eval()
mlp = AestheticMLP(input_size=768)            # 4-layer MLP, defined inline below
mlp.load_state_dict(torch.load(MLP_WEIGHTS, map_location=DEVICE))
mlp.to(DEVICE).eval()

app = FastAPI()

@app.get("/health")
def health() -> dict:
    return {"ok": True, "device": DEVICE, "model": "ViT-L-14 + LAION MLP"}

@app.post("/score")
async def score(image: UploadFile = File(...)) -> dict:
    try:
        raw = await image.read()
        img = Image.open(io.BytesIO(raw)).convert("RGB")
    except Exception as exc:
        raise HTTPException(400, f"could not decode image: {exc}")
    with torch.no_grad():
        tensor = preprocess(img).unsqueeze(0).to(DEVICE)
        feats = clip_model.encode_image(tensor)
        feats = feats / feats.norm(dim=-1, keepdim=True)
        score = float(mlp(feats).cpu().item())
    return {"score": score}
```

`AestheticMLP` is the standard 4-layer MLP from
`christophschuhmann/improved-aesthetic-predictor` (vendored in `server.py`
-- ~30 lines). MLP weights `sac+logos+ava1-l14-linearMSE.pth` (~6MB)
downloaded by `vast_setup.sh`.

### Three properties this exercises that offline tests can't

1. **Real Flux output shape**: 1024x1024 RGB PNG, ~150-300KB on disk.
   Verifies our workflow JSON is actually valid against live ComfyUI (the
   offline `httpx.MockTransport` tests only verify our request shape).
2. **Real LAION score distribution**: confirms scores land in the 4-7 range
   Cask's atmospheric prompts target. If they're all <3 or all >9, our
   threshold needs re-tuning before Sessions 7+.
3. **Selection rule on real scores**: with three real candidates having
   three distinct scores, we see whether the highest-passing candidate
   actually *looks* better than the lowest. A spot human-eye review at end
   of smoke validates the LAION model is doing useful work.

## 6. Error handling

| Failure mode | Where | Behaviour |
|---|---|---|
| `aesthetics.host` missing/empty | `RemoteAestheticScorer.__init__` | Raises `ValueError` immediately with pointer to `.env` / `secrets.yaml`. Fail-fast at Stage entry, before any GPU calls. |
| score_server unreachable (connection refused, DNS fail) | `httpx.AsyncClient.post` | Raises `httpx.ConnectError`. Caught at per-candidate boundary in `generate_for_scene` -> `score=0.0`, `scoring_succeeded=False`. Logged WARN. |
| score_server slow (>30s) | `httpx.AsyncClient` timeout | Raises `httpx.TimeoutException`. Same treatment as connection error. The 30s default is generous (LAION inference is ~50-200ms; timeout exists to defeat hung sockets). |
| score_server returns non-200 | `resp.raise_for_status()` | Raises `httpx.HTTPStatusError`. Caught per-candidate. |
| score_server returns NaN/inf | `RemoteAestheticScorer.score` | Raises `ValueError`. Caught per-candidate (the existing `_is_finite` guard in `generate_for_scene` would also handle a NaN return value, so we could let NaN through -- but raising is more honest about the upstream bug). |
| score_server returns malformed JSON | `body["score"]` `KeyError` / `resp.json()` `JSONDecodeError` | Caught per-candidate, `score=0.0`, `scoring_succeeded=False`, WARN. |
| Image upload fails mid-stream (file vanished, disk error) | `image_path.open` | Raises `FileNotFoundError` / `OSError`. Caught per-candidate. |
| **All 3 candidates' scoring threw (infra failure)** | `generate_for_scene` after candidate loop | `not any(scoring_succeeded)` -> raise `KeyframeGenerationError`. Stage halts; StageRun marked FAILED. **NEW in S6.1.** |
| **All 3 candidates scored but none passed gates (content failure)** | `generate_for_scene` selection | `selected_via_fallback=True`, fallback to highest-scored among scored set (not raw candidate 0). Stage continues. |
| ComfyUI host wrong/unreachable | `HttpComfyClient.generate_image` | Pre-existing S6 behaviour: per-candidate exception -> score=0.0, anatomy=False, eligible via fallback only. If all 3 throw, `KeyframeGenerationError` halts the stage. |
| Vast.ai instance dies mid-smoke | network errors on subsequent calls | Per-candidate exceptions cascade. Atomic per-scene `story.save` means scenes already complete are preserved. Re-rent + re-run picks up from `keyframe_path is not None` resume check. No work lost beyond the in-flight scene. |
| `.env` not present + no env vars set | `Config()` load | Defaults `comfyui.host`, `aesthetics.host` to empty/localhost. `RemoteAestheticScorer.__init__` then raises `ValueError`. CLI catches and prints "set PLATINUM_AESTHETICS_HOST in .env -- see runbook." |
| `--scenes "0,7,15,abc"` parse failure | CLI typer callback | Raises `typer.BadParameter` with "must be comma-separated integers". Exit 2. |
| `--scenes "0,7,99"` references missing scene | CLI before stage entry | Validates `max(scene_filter) < len(story.scenes)`; raises `typer.BadParameter` with story's actual scene count. Exit 2. |
| Story has no `visual_prompts` COMPLETE StageRun | CLI before stage entry | Print `[red]story <id> has no completed visual_prompts; run 'platinum adapt' first[/red]`. Exit 1. |

### How `test_score_server.py` mocks torch + open_clip

`server.py` imports `torch` and `open_clip` at module load and calls
`clip_model.encode_image` at request time. Local pytest doesn't have GPU
torch. Solution: `tests/unit/test_score_server.py` uses `sys.modules`
insertion *before* importing `server`:

```python
def _install_fake_torch_and_clip(monkeypatch):
    fake_torch = types.ModuleType("torch")
    fake_torch.cuda = types.SimpleNamespace(is_available=lambda: False)
    fake_torch.no_grad = lambda: contextlib.nullcontext()
    fake_torch.load = lambda *a, **k: {}
    monkeypatch.setitem(sys.modules, "torch", fake_torch)

    fake_open_clip = types.ModuleType("open_clip")
    fake_clip_model = MagicMock()
    fake_clip_model.encode_image.return_value = MagicMock(
        norm=lambda *a, **k: MagicMock(),
        __truediv__=lambda self, other: MagicMock(),
    )
    fake_open_clip.create_model_and_transforms = lambda *a, **k: (
        fake_clip_model, None,
        lambda img: MagicMock(unsqueeze=lambda d: MagicMock(to=lambda dev: MagicMock())),
    )
    monkeypatch.setitem(sys.modules, "open_clip", fake_open_clip)
```

Then the test imports `scripts.score_server.server` (added to `sys.path`
via a conftest fixture) and uses FastAPI's `TestClient`. The MLP itself is
a tiny class defined inline in `server.py`; tests can patch `server.mlp` to
a `MagicMock` returning a tensor whose `.cpu().item()` is a fixed score.

This is heavier mocking than the rest of the codebase. **Justified**
because the alternative -- installing `torch` (~2GB) into the local venv
just to test a 30-line FastAPI handler -- is wildly disproportionate, and
the score_server's *integration* test already happens at the end of the
session via the live smoke run on the box.

### `test_aesthetics.py` `RemoteAestheticScorer` mock pattern

```python
async def test_remote_scorer_happy_path():
    captured = {}
    async def handler(request: httpx.Request) -> httpx.Response:
        captured["url"] = str(request.url)
        captured["content"] = request.content
        captured["headers"] = dict(request.headers)
        return httpx.Response(200, json={"score": 6.42})

    transport = httpx.MockTransport(handler)
    scorer = RemoteAestheticScorer(host="http://test:8189", transport=transport)
    try:
        score = await scorer.score(Path("tests/fixtures/keyframes/candidate_0.png"))
    finally:
        await scorer.aclose()

    assert score == 6.42
    assert captured["url"] == "http://test:8189/score"
    assert b"candidate_0.png" in captured["content"]      # multipart filename
    assert captured["headers"]["content-type"].startswith("multipart/form-data")
```

Pattern matches `HttpComfyClient` exactly. Nothing new for the test author
to learn.

### TDD rhythm

Each task in the implementation plan ends with a green `pytest -q`. Same
Session 4-6 cadence: (1) failing test, (2) run-see-fail, (3)
minimum-implementation, (4) run-see-pass, (5) commit.

### Quality bar (after task N)

- `pytest -q` shows ~254 tests, 0 fail, 0 skip.
- `ruff check src tests scripts` clean.
- `mypy src` clean (no new errors; pre-existing 2 stay deferred per S5/S6 review).
- Live smoke run produces 9 PNGs on disk (3 scenes x 3 candidates) +
  correctly populated `story.json` for scenes 0/7/15.
- Manual eye-check: open the 9 PNGs in a viewer; confirm Cask's
  atmospheric prompts produce vaguely-on-theme images. (Pure sanity --
  not a test gate.)

### What is deliberately NOT tested

- **Real LAION model accuracy.** We trust the upstream
  `improved-aesthetic-predictor` weights. We verify the model loads,
  scores, and returns a float -- not that it correctly ranks aesthetics.
- **score_server under load / concurrent requests.** Single-worker
  serialised; if we ever go concurrent, that's a follow-up session.
- **vast.ai SDK / `vastai` CLI.** Rental is a manual runbook step.
  Automating the rental is out of scope.
- **Cross-instance teardown idempotency.** Tear down is a single
  `vastai destroy <id>`; if it fails we re-run.

## 8. Live smoke runbook (outline)

The runbook itself becomes `docs/runbooks/vast-ai-keyframe-smoke.md`.
Section structure:

```
1. Prerequisites

6. Wire platinum to the box
```

### Two intentional half-measures worth flagging

1. **`scoring_succeeded` field on `KeyframeReport`** (decided in Section
   6) is a half-measure: we use it to distinguish infra failure from
   content failure, but we DON'T use it to retry a failed scoring call. A
   future enhancement could retry score_server calls with backoff (httpx
   already has retry primitives). For now: per-candidate failure =
   `score=0.0`, `scoring_succeeded=False`. Single-attempt is fine for
   S6.1's smoke; live retries land if/when uvicorn flakiness becomes a
   real problem.

2. **Manual eye-check at runbook step 9** is the only validation that
   LAION model output is "useful." We have zero automated test that the
   upstream `improved-aesthetic-predictor` weights produce sensible
   scores. If they don't, the smoke run still "passes" with all-fallback
   selections. The eye-check catches this. A real fix (a regression suite
   of known-good and known-bad images with expected score ranges) is out
   of scope.

## 10. Lessons carried in (and one new one)

1. **Late binding for testability.** `RemoteAestheticScorer.__init__(transport=None)`
   resolves transport at call time so tests inject `httpx.MockTransport(handler)`.
   Same pattern as `HttpComfyClient(transport=None)` (S6),
   `mp_hands_factory=None` (S5/S6), `asyncio.sleep` monkeypatch path (S4),
   `subprocess.run` injection (S3).

2. **Recorder/Fake protocol pattern.** `AestheticScorer` Protocol now has
   three implementations: `FakeAestheticScorer` (single fixed score),
   `MappedFakeScorer` (per-path map), `RemoteAestheticScorer` (real
   HTTP). Production injects real; tests inject either Fake variant.
   Mirrors `ComfyClient` / `Recorder` / `AestheticScorer` from S4-S6.

3. **Pure-core / impure-shell at the Stage boundary.**
   `generate_for_scene` and `generate` stay pure async functions;
   `KeyframeGeneratorStage` remains the impure shell that pulls deps
   from `ctx`. S6.1 only adds: shell-side
   `try/finally: await scorer.aclose()` for the new lifecycle.

4. **MockTransport beats Fake for HTTP-shape verification.** S6 proved
   this with `HttpComfyClient`; S6.1 reuses verbatim for
   `RemoteAestheticScorer` and `score_server`. The Fake covers protocol
   surface; MockTransport covers wire shape; FastAPI TestClient covers
   server-side handler. Three lenses, no overlap.

5. **Per-stage atomic save.** `story.save` uses tmp + os.replace;
   per-scene save inside `generate` gives free crash-resilience. Already
   proven through Sessions 3-6. Live smoke is the first test that
   exercises this against a non-trivial latency profile (real Flux is
   ~30-60s/candidate; a mid-scene crash means we lose at most one
   scene's work).

6. **Subagent scope drift on "chore" / "sweep" tasks.** Task closeouts
   during S6.1 implementation will explicitly call out: do not commit
   `data/stories/*` (real keyframe artifacts will be ~150-300KB x 9 =
   ~1.5-3 MB of PNGs -- easy to accidentally commit), do not commit
   `.env` or live host URLs, paste actual `pytest -q` and `ruff check`
   output rather than claiming clean.

### One new lesson S6.1 will produce

7. **Live-smoke runbook discipline (NEW).** Session 6.1 establishes the
   pattern for every future GPU-touching session (S8 video, S9 upscale,
   S10 voice, S13 mix). The `docs/runbooks/vast-ai-keyframe-smoke.md`
   template -- prerequisites -> rent -> provision -> launch -> verify ->
   wire -> dry-run -> run -> eye-check -> tear down -> troubleshooting --
   gets cloned for each subsequent live session. The "troubleshooting"
   section, populated *during* the live run, is the high-value artifact:
   each anti-pattern caught becomes a future-you bullet that saves 30+
   min on the next rental.
