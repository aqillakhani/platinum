## 1. Goal & scope

Session 6.1 shipped the live keyframe pipeline end-to-end against a vast.ai
RTX 4090 32GB box, but the FP8 + cpu-vae + lowvram flag stack we needed to
fit Flux1-dev fp16 + t5xxl_fp16 in 32GB system RAM produced perceptually
degenerate output: all 9 candidates across Cask of Amontillado scenes
1/8/16 came back essentially black (mean RGB 0-3 of 255), and
LAION-Aesthetics scored them 3.9-4.6 -- below the 6.0 track threshold,
which correctly rejected them in the eligibility filter, but the
"highest-scored among scored" fallback rule still selected and persisted
them. Session 6.2 fixes the model-layer collapse and closes the perceptual
safety-net hole.

By end of session:

- vast.ai box class is **A6000 48GB VRAM + >=64GB system RAM**;
  `launch_comfyui.sh` carries only `--mmap-torch-files`. No FP8 flags, no
  `--cpu-vae`, no `--lowvram`. Pure fp16 + GPU VAE.
- `config/workflows/flux_dev_keyframe.json` retuned: `sampler_name:
  dpmpp_2m`, `scheduler: karras`, `steps: 30`, `cfg: 3.5`. 1024x1024
  unchanged.
- `src/platinum/utils/validate.py` exports `check_image_brightness`. Wired
  into `generate_for_scene` BEFORE the LAION call (saves the network
  round-trip on rejected candidates) and into the eligible + fallback
  filters in `keyframe_generator.py`. `KeyframeReport` gains
  `brightness_passed: list[bool]`.
- All five track YAMLs gain `brightness_floor_mean_rgb` under
  `quality_gates`. Atmospheric_horror's permissive 20.0 lets canonical
  chiaroscuro pass; brighter tracks (children's fables 80, slice-of-life
  60, folktales 60, sci-fi 40) reject darker outputs as miscalibrated.
- New `scripts/keyframe_quality_smoke.py` driver runs `generate_for_scene`
  against synthetic Scene objects -- used for the bright-probe smoke step
  with hand-authored prompts (no Claude / no Story setup).
- `docs/runbooks/vast-ai-keyframe-smoke.md` updated: A6000 search filter,
  flag-stack diff, dual-step smoke (regression + bright probe), no
  teardown (box stays up for Session 7).
- 254 -> ~262 tests (+8 net new). 0 fail, 0 skip. ruff + mypy bar
  preserved.
- Live smoke produces **9 Cask PNGs (scenes 1/8/16) with mean_rgb >= 20
  each** + LAION 5.5-7.5, eye-check confirms catacomb / wine cellar /
  candlelit identity. Plus **6 bright-probe PNGs** (2 prompts x 3
  candidates) with mean_rgb >= 100 each + LAION 5.5-7+.

This session does NOT ship IP-Adapter FaceID, ControlNet Depth,
`flux_pro_api`, or Wan 2.2. Those land in Sessions 6.3 / 6.4 / 8.

## 2. Decisions (four brainstorming forks)

| # | Question | Decision |
|---|---|---|
| 1 | Quality fix only, or bundle in IP-Adapter / ControlNet wiring? | **Quality fix only.** The black-image bug is fundamental -- IP-Adapter on broken Flux gives broken Flux + reference leak. Bundling makes the smoke a multi-variable experiment with no isolation. Promote IP-Adapter to Session 6.3, ControlNet Depth to 6.4. Provisioning amortization argument is weak when re-rental for S7's full Cask is already on the calendar. |
| 2 | Quality fix approach | **Bigger box (A6000 48GB) + drop ALL FP8 / cpu-vae / lowvram flags** + sampler retune (`dpmpp_2m` / `karras` / 30 steps) **+ darkness-floor pre-check**. The S6.1 root cause was system RAM, not VRAM (Flux1-dev fp16 24GB + t5xxl fp16 9GB = 33GB at load, overflowed 32GB RAM cap). A6000 box class with >=64GB system RAM eliminates the constraint cleanly. Sampler retune is the well-trodden Flux community config (was on ComfyUI's `euler`/`simple`/20 default). Darkness-floor closes the LAION-can't-reject-garbage hole permanently as defense-in-depth. Rejected option D (raise LAION threshold) -- the threshold gate already engaged at 6.0; the fallback rule bypassed it. Rejected option B (Comfy-Org pre-quantized FP8) -- untested for our prompt distribution; we'd be the test case. Rejected option E (drop only `--cpu-vae`) -- might OOM at 20GB peak on 24GB card and might not even be the root cause. |
| 3 | Tear down at end again, or leave the box up? | **Leave up for Session 7.** Session 7's first task is full Cask 16-scene run; runs immediately after S6.2. Saves 30-45 min provisioning. Document explicitly: S7's runbook starts at "verify endpoints + dry-run" with services already live; teardown reminder lands in S7's design. |
| 4 | Same Cask story, or different? | **Cask 1, 8, 16 (regression) + 2 hand-authored bright probe prompts.** Cask is the direct regression test on the original failure (all three scene prompts are explicitly chiaroscuro). Bright probe (sunlit Tuscan vineyard, snowy alpine peak at dawn) tests the brightness range no Cask scene exercises -- proves the fix isn't dark-only and the brightness gate calibrates correctly across the distribution other tracks (folktales, sci-fi, slice-of-life, children's fables) will bring later. Hand-authored avoids Claude API spend (~$1.50) and Story-setup overhead; ~30-line driver script reuses `generate_for_scene` directly. |

## 4. Components

### 4a. `check_image_brightness`

```python
def check_image_brightness(
    path: Path,
    *,
    min_mean_rgb: float = 20.0,
) -> CheckResult:
    """Reject perceptually degenerate (near-black) images.

    LAION-Aesthetics v2 doesn't penalise low-content imagery (a fully black
    PNG scores 3.9-4.6). This primitive provides a hard floor independent
    of the aesthetic scorer -- catches FP8 + cpu-vae collapse, VAE decode
    bugs, and similar model-layer failures.

    Mean is computed over all RGB pixels (alpha ignored); fully black image
    has mean_rgb=0; pure white has mean_rgb=255.
    """
    try:
        from PIL import Image
    except ImportError:
        return CheckResult(
            passed=True,
            metric=0.0,
            threshold=min_mean_rgb,
            reason="skipped: PIL unavailable",
        )
    try:
        with Image.open(path) as img:
            arr = np.asarray(img.convert("RGB"), dtype=np.float64)
        mean_rgb = float(arr.mean())
    except Exception as exc:  # noqa: BLE001
        return CheckResult(
            passed=False,
            metric=0.0,
            threshold=min_mean_rgb,
            reason=f"failed to read image: {exc!r}",
        )
    if mean_rgb >= min_mean_rgb:
        return CheckResult(
            passed=True,
            metric=mean_rgb,
            threshold=min_mean_rgb,
            reason=f"ok (mean_rgb={mean_rgb:.1f} >= min_rgb={min_mean_rgb:.1f})",
        )
    return CheckResult(
        passed=False,
        metric=mean_rgb,
        threshold=min_mean_rgb,
        reason=f"image too dark (mean_rgb={mean_rgb:.1f} < min_rgb={min_mean_rgb:.1f})",
    )
```

Pattern matches `check_hand_anomalies` exactly: defensive ImportError
handler returning `passed=True, reason="skipped: ..."` so missing PIL
doesn't halt the pipeline (consistent with S6.1's mediapipe drift fix).

### 4d. `scripts/keyframe_quality_smoke.py`

```python
"""Bright-probe smoke driver: feeds a single hand-authored prompt through
generate_for_scene against a synthetic Scene, prints per-candidate stats.

Bypasses Story / StageRun / orchestrator -- testing only the gen + score
+ select layer with arbitrary prompts. Used by the S6.2 runbook's bright-
probe step + future sessions when ad-hoc prompts need quick validation.
"""
import asyncio
import argparse
from dataclasses import dataclass, field
from hashlib import sha256
from pathlib import Path
from typing import Any

from platinum.config import Config
from platinum.utils.aesthetics import RemoteAestheticScorer
from platinum.utils.comfyui import HttpComfyClient
from platinum.pipeline.keyframe_generator import generate_for_scene


@dataclass
class _SyntheticScene:
    index: int
    visual_prompt: str
    negative_prompt: str = ""
    keyframe_path: Path | None = None
    keyframe_candidates: list[Path] = field(default_factory=list)
    keyframe_scores: list[float] = field(default_factory=list)
    validation: dict[str, Any] = field(default_factory=dict)


async def run(
    *, prompt: str, label: str, negative: str, track: str, output_dir: Path
) -> None:
    cfg = Config()
    track_cfg = cfg.track(track)
    track_visual = dict(track_cfg.get("visual", {}))
    quality_gates = dict(track_cfg.get("quality_gates", {}))

    scene_index = int.from_bytes(sha256(label.encode()).digest()[:2], "big") % 10000
    scene = _SyntheticScene(
        index=scene_index, visual_prompt=prompt, negative_prompt=negative
    )
    scene_dir = output_dir / label
    scene_dir.mkdir(parents=True, exist_ok=True)

    comfy = HttpComfyClient(host=cfg.settings["comfyui"]["host"])
    scorer = RemoteAestheticScorer(host=cfg.settings["aesthetics"]["host"])
    try:
        report = await generate_for_scene(
            scene,
            track_visual=track_visual,
            quality_gates=quality_gates,
            comfy=comfy,
            scorer=scorer,
            output_dir=scene_dir,
            config_dir=cfg.config_dir,
        )
    finally:
        await scorer.aclose()

    # Print rich-formatted per-candidate table; details left to impl plan.
    print(f"label={label} scene_index={scene.index}")
    for i, path in enumerate(report.candidates):
        print(
            f"  cand {i}: score={report.scores[i]:.2f}  "
            f"bright={report.brightness_passed[i]}  "
            f"anat={report.anatomy_passed[i]}  "
            f"sel={'yes' if i == report.selected_index else '-'}  "
            f"fb={'yes' if (i == report.selected_index and report.selected_via_fallback) else '-'}"
        )


def _main() -> None:
    parser = argparse.ArgumentParser()
    parser.add_argument("--prompt", required=True)
    parser.add_argument("--label", required=True)
    parser.add_argument("--negative", default="")
    parser.add_argument("--track", default="atmospheric_horror")
    parser.add_argument("--output-dir", type=Path, default=Path("/tmp/smoke"))
    args = parser.parse_args()
    asyncio.run(
        run(
            prompt=args.prompt,
            label=args.label,
            negative=args.negative,
            track=args.track,
            output_dir=args.output_dir,
        )
    )


if __name__ == "__main__":
    _main()
```

### Three quality validation surfaces

1. **Mean-RGB hard floor**: every persisted candidate has measurable
   non-zero content.
2. **LAION score distribution**: with the FP8 collapse fixed, scores
   should land in the trained 5-7 range Cask atmospherics target. If
   they're <5 across the board, sampler retune wasn't enough; if >8,
   threshold may be too lax.
3. **Eye-check distinct candidates**: 3 different seeds produce 3
   visibly different compositions (different framing, lighting accents,
   character poses). If they collapse to look-alikes, deterministic
   seeding may not be wired through the workflow correctly.

## 6. Error handling

| Failure mode | Behaviour |
|---|---|
| Single candidate fails brightness floor | `brightness_passed[i]=False`, `scoring_succeeded[i]=False`, `score=0.0`, `anatomy_passed[i]=False`. Skip LAION call (saves the round-trip). Logged WARN with `bright.reason`. Other candidates continue normally. |
| All 3 candidates fail brightness floor | `KeyframeGenerationError` -- halt stage. Treated identically to S6.1's "all candidates failed scoring" infra-failure path: best-effort `story.save(...)` for prior-scene checkpointing, then re-raise. User sees `[red]` CLI message + failed scene index, diagnoses (likely: model regression, weight corruption, or VAE failure), re-runs. |
| Brightness + scoring partial mix | Eligibility filter ANDs brightness_passed; fallback uses (`scoring_succeeded AND brightness_passed`) set. Brightness-failing candidates are NEVER picked, even in fallback. |
| Track YAML missing `brightness_floor_mean_rgb` | Default 20.0 (matches atmospheric_horror; permissive). Logged INFO once per stage run. |
| `check_image_brightness` ImportError on PIL | Returns `passed=True, reason="skipped: PIL unavailable"`. Pipeline continues. Mirrors S6.1's mediapipe defensive handler. |
| `check_image_brightness` PIL decode error | Returns `passed=False, reason="failed to read image: ..."`. Treated same as brightness-floor failure -- candidate skipped. |
| A6000 box rental returns "no offers" | Runbook fallback ladder: A6000 Ada -> A100 40GB -> A100 80GB -> H100 80GB. Stop at first available. Document cost progression in runbook. |
| Re-rented box has different IP after stop/start | Update `.env` host vars, no other change. (Already validated in S6.1.) |
| Pre-existing failure modes (ComfyUI unreachable, score_server crash, etc.) | All preserved from S6.1's matrix; brightness gate is purely additive. |

### Two intentional half-measures worth flagging

1. **Brightness floor is mean-RGB only** -- a uniform mid-gray (mean=127)
   would pass even though it's perceptually empty. If we hit a
   "low-content but technically bright" failure mode in S7+, the gate can
   be upgraded to pixel-variance / entropy / edge-density. For S6.2's
   black-PNG failure mode, mean-RGB is the right tool.

2. **Bright probe is 2 prompts only.** If the fp16 fix passes Cask
   (atmospheric horror) and these 2 hand-authored bright prompts but the
   actual variance across all five tracks (folktales, sci-fi, slice-of-
   life, children's fables) reveals different failure modes when those
   tracks run in S12+, we'll add per-track probe prompts then. For S6.2,
   2 prompts that span the brightness range we know exists is enough.

## 10. Lessons carried in (and one new one)

1. **Late binding for testability.** `RemoteAestheticScorer.transport=None`,
   `mp_hands_factory=None`, `check_image_brightness` direct (no factory
   needed -- pure function). Same pattern as S3-S6.1.

2. **Recorder/Fake protocol pattern.** No new Protocols in S6.2;
   `check_image_brightness` is pure (not behind an interface).

3. **Pure-core / impure-shell at the Stage boundary.** Brightness gate
   lives in `generate_for_scene` (pure async). Stage shell unchanged
   from S6.1.

4. **MockTransport / FastAPI TestClient seam unchanged.** S6.2 doesn't
   add HTTP surface area.

5. **Per-stage atomic save.** Brightness halt path uses the same
   best-effort `story.save(...)` before re-raise as S6.1's scoring halt.

6. **Subagent scope drift on chore tasks.** Task closeouts during S6.2
   implementation will explicitly call out: do NOT commit
   `data/stories/*` (live regenerated keyframes will be ~150-300KB x 15
   = ~2-4 MB easy to accidentally commit), do NOT commit `.env` or live
   host URLs, paste actual `pytest -q` and `ruff check` output rather
   than claiming clean.

7. **Final cumulative review catches gaps individual task reviews miss.**
   S6.1's lesson stays mandatory. End every implementation phase with
   one final "does the system work end-to-end, not just task-by-task?"
   pass.

### One new lesson S6.2 will produce

8. **NEW: Perceptual safety nets need cheap pre-filters.** LAION-Aesthetics
   is trained on a different distribution than "is this image perceptually
   degenerate?" -- it ranks aesthetics on the assumption inputs are
   *images*. A black PNG isn't an image to LAION's training distribution;
   it gets a roughly random middle score (3.9-4.6 in our case). The
   correct architecture is a cheap hard pre-filter that gates the
   sophisticated scorer: `mean_rgb >= floor` here, but the principle
   generalises -- before any score-based gate, run a "is the input even
   in-distribution for the scorer" check. Document expects this pattern
   to recur in S8 (motion-flow score on still frames is meaningless --
   gate on `is_video_not_still`), S9 (PSNR on output of a failed upscale
   is meaningless -- gate on `output_size_increased`), S13 (LUFS on a
   silent track is meaningless -- gate on `audio_present`).
