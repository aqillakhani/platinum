## Risks and mitigations

1. **Bible LLM hallucinates facts not in story.** Mitigation: bible
   prompt instructs Opus to ground every claim in `narration_text` or
   `adapted source`; recorder fixture is reviewed by hand before
   commit; later sessions can add a verification pass.

2. **Rewriter still drops characters despite the post-condition.**
   Mitigation: post-condition raises → orchestrator's existing
   single-retry path runs the call again. If it fails twice, halt with
   a clear error — better than silent corruption.

3. **Brightness floor too restrictive, hurts intentional darkness.**
   Mitigation: floor levels are `low|medium|high`; `low` permits Flux
   chiaroscuro down to mean RGB ~5–10 (current track floor). Only
   `medium`/`high` fire the strict guardrail. Bible prompt teaches Opus
   to pick `low` for night/intimate and `medium`/`high` only when an
   active light source is in scene.

4. **Bible cost grows with story length.** Mitigation: per-track
   `max_tokens` caps it. Cask is 16 scenes, Opus 4.7 input ~12K, output
   ~6K. For 32-scene stories the prototype suggests ~$0.55 per bible —
   still negligible vs. GPU rental.

5. **Tests over-mock and miss real-world rewriter failure.**
   Mitigation: S8.B.5's recorder fixture is a real Sonnet call against
   the recorded bible — true end-to-end. The verify rental on Cask is
   the final integration test before closure.
