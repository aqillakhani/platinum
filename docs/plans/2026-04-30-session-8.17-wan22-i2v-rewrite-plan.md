# S8.17 — Wan 2.2 I2V workflow rewrite (implementation plan)

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Rewrite `config/workflows/wan22_i2v.json` to a working 11-node Wan 2.2
14B I2V graph using the Wan 2.1 VAE and a two-sampler MoE chain, plus the
supporting `inject_video` / `vast_setup.sh` / preflight changes, so Phase 2
can re-rent vast.ai with confidence the workflow will run.

**Architecture:** Two-expert MoE chain (HIGH expert end_step=15, LOW expert
start_step=15 with `samples` handoff), single `WanVideoImageToVideoEncode`
feeding both samplers, Wan 2.1 VAE shared across encode + decode,
`VHS_VideoCombine` for MP4 output. `inject_video` gains a list-aware role
resolver so the same seed lands on both samplers; preflight gains a 7th
check for the VideoHelperSuite extension.

**Tech Stack:** Python 3.11, pytest, ComfyUI-WanVideoWrapper (Kijai),
ComfyUI-VideoHelperSuite (Kosinkadink), bash provisioning.

**Predecessor design:** `docs/plans/2026-04-30-session-8.17-wan22-i2v-rewrite-design.md`
(commit `0357596`). All `WHY`s and root-cause analysis live there.

def _resolve_role_ids(workflow: dict[str, Any], role: str) -> list[str]:
    """Resolve a role to a list of node IDs.

    Single-string roles wrap to a one-element list; list-valued roles pass
    through. Used by `inject_video` for the `seed` role, which on the Wan
    2.2 14B I2V workflow points at two samplers (HIGH + LOW) that must
    share a coupled seed for determinism across the MoE handoff.
    """
    roles = workflow.get("_meta", {}).get("role", {})
    if role not in roles:
        raise KeyError(f"workflow _meta.role missing required role: {role!r}")
    val = roles[role]
    return [val] if isinstance(val, str) else list(val)
```

def _check_videohelpersuite_extension_dir(comfyui_dir: Path) -> tuple[bool, str]:
    """Check that ComfyUI-VideoHelperSuite is present under custom_nodes/.

    Folder-only check (no Python import). VHS registers nodes via
    __init__.py at ComfyUI startup; if the folder is there and
    requirements.txt was pip-installed (vast_setup.sh's responsibility),
    the VHS_VideoCombine node will be registered. Used by the Wan 2.2
    I2V workflow's video_out role.
    """
    vhs_dir = Path(comfyui_dir) / "custom_nodes" / "ComfyUI-VideoHelperSuite"
    if not vhs_dir.is_dir():
        return False, f"VideoHelperSuite extension missing at {vhs_dir}"
    return True, f"VideoHelperSuite present at {vhs_dir}"
```

The signature takes a `comfyui_dir` parameter (matching the
`tmp_path`-driven test) so tests can stub the location; the production
caller in `main()` will pass `Path("/workspace/ComfyUI")`.

If `Path` isn't already imported at the top of the module, add
`from pathlib import Path`.
