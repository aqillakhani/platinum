## 1. Goal & scope

Session 6.2 Phase 2 surfaced that the keyframe quality issue was NOT FP8
quantization -- it was the workflow itself. Our `config/workflows/flux_dev_keyframe.json`
is missing the `FluxGuidance` and `ModelSamplingFlux` nodes that Flux Dev (a
guidance-distilled model) requires to receive guidance via the conditioning
pathway. Our current `KSampler.cfg=3.5` applies classic CFG, which the distilled
model wasn't trained for, producing "mood reductions" of dense prompts:
atmosphere/colors render but objects/subjects collapse into impressionistic
blobs. Live evidence on A6000 fp16: the Tuscan bright probe (with track aesthetic
prefix dropped, mean RGB ~100, LAION 4.3) showed yellow sky over dark ground
**with no vineyard, cypress, or farmhouse rendered**. Cask scene 1 chiaroscuro
showed faint amber smudge in 99% black canvas with **no nobleman, goblet, or
furniture**.

Session 6.3 fixes the workflow, supplements the brightness floor with a
subject-detection gate (catches "high-brightness mood-only" failures the floor
misses), and removes the track aesthetic prefix that was double-counting
darkness signals.

By end of session:

- `config/workflows/flux_dev_keyframe.json` carries the full ComfyUI Flux Dev
  reference pattern: `KSampler.cfg=1.0`, plus `FluxGuidance(guidance=3.5)`
  inserted between the positive `CLIPTextEncode` and the `KSampler.positive`
  slot, plus `ModelSamplingFlux(max_shift=1.15, base_shift=0.5,
  width=1024, height=1024)` inserted between `UNETLoader` and `KSampler.model`.
  New `_meta.role` tags `flux_guidance` and `model_sampling_flux` are
  registered. The full reference is the **shipped state** -- if Phase 2's
  Probe A (cfg=1.0 alone, no node additions) succeeds, we still ship the full
  reference workflow because cfg=1.0 with no FluxGuidance is "no guidance
  signal at all" and is not a config we'd want to leave in production.
- `src/platinum/utils/validate.py` exports `check_image_subject` -- edge-density
  primitive based on OpenCV Canny. Returns `CheckResult` with the same
  contract as `check_image_brightness`.
- `src/platinum/pipeline/keyframe_generator.py`: subject gate runs AFTER
  brightness, BEFORE the LAION call (saves the network round-trip on
  brightness-failing OR subject-failing candidates). `KeyframeReport` gains
  `subject_passed: list[bool]`. Eligibility filter ANDs subject_passed.
  Fallback excludes subject-failing candidates (mirrors brightness exactly).
  Halt-on-all-fail rule extended: `not any(subject_passed)` -> raise
  `KeyframeGenerationError`.
- The track aesthetic prefix at line 105 is REMOVED. `inject(workflow, prompt=...)`
  now receives `scene.visual_prompt` directly. `track_visual["aesthetic"]` is
  no longer read; the YAML keys remain (no schema churn) but the field is
  effectively unused. Documented as a "deprecated, may be removed in a future
  cleanup commit" note in the YAML files.
- All five track YAMLs gain `subject_min_edge_density` under `quality_gates`.
  Initial values: atmospheric_horror 0.020, folktales / scifi / slice-of-life
  0.030, childrens_fables 0.040. These are best-guess defaults; calibrated
  against live data in Phase 2.
- `scripts/preflight_check.py` runs after `vast_setup.sh` provisioning. Five
  fail-fast checks (HF token resolve, workflow JSON validity, ComfyUI alive,
  score-server alive, workflow signature print). Catches the "whoami passes
  but token lacks gated access" gotcha + the "stale code on box" gotcha.
- `scripts/reset_scene_keyframes.py` is a small helper to clear keyframe state
  for specified scenes -- needed because the orchestrator skips scenes whose
  `keyframe_path` is already set, and Phase 2's validation step requires
  re-running scenes 1/8/16 from scratch after the workflow rebuild.
- `scripts/keyframe_quality_smoke.py` (existing from S6.2) is unchanged at the
  CLI surface -- it picks up the rebuilt workflow JSON automatically.
- `docs/runbooks/vast-ai-keyframe-smoke.md` updated for the two-rental pattern,
  pre-flight step, and Phase 2's probe sequence (cfg=1.0 alone -> full
  reference if needed -> Cask validation).
- 274 -> ~300 tests (+~26 net new). 0 fail, 0 skip. ruff + mypy bar preserved.
- Phase 2 live smoke produces:

## 2. Decisions (six brainstorming forks)

| # | Question | Decision |
|---|---|---|
| 1 | Workflow rebuild order -- diagnostic isolation or known-good first? | **C: sequential ablation in a single rental.** Probe A (cfg=1.0 alone) first; if it produces detailed subjects, Probe A wins. If it doesn't, Probe B (full reference rebuild with FluxGuidance + ModelSamplingFlux) follows in the same rental. Both probes use `keyframe_quality_smoke.py` against Tuscan + Cask scene 1. Marginal cost between A-only and A+B is ~$0.05; max info per probe. **Note**: regardless of which probe wins on the live eye-check, S6.3 ships the FULL REFERENCE workflow (cfg=1.0 + both nodes). Probe A is a diagnostic step, not a "minimum viable workflow" candidate -- "cfg=1.0 with no FluxGuidance" is not a configuration we want to operate in production. |
| 2 | Subject-detection gate primitive | **A: edge density via OpenCV Canny.** Most directly addresses the observed failure mode (color field with no subject boundaries). Cheapest primitive that's not generic -- a noisy garbage image would also have high edge density (entropy would too), but neither edge density nor entropy can distinguish "noisy garbage" from "real scene", and that's not the failure mode we've actually observed on live A6000. We ship A in S6.3. If S7's full Cask run surfaces a "high-edge garbage" failure mode, we add a complementary primitive then. Defer-to-S6.4 was explicitly REJECTED -- the user emphasized "quality over everything"; the gate is cheap defense-in-depth that catches future regressions. |
| 3 | Track aesthetic prefix policy | **A: drop entirely.** The visual_prompts S4 stage already incorporates track aesthetic per-scene (it was instructed to). Prefix at `keyframe_generator.py:105` double-counts and actively contradicts for non-aligned content (bright probe in horror track). Drop is simplest and removes a confound from the workflow probe diagnostic. YAML schema unchanged (key stays, just unread); formal removal is a separate cleanup pass deferred to S7+. |
| 4 | Validation scope | **B: Tuscan + Cask 1, 8, 16.** Three darkness levels stress-test the workflow without burning budget on full 16-scene if the first three already fail. Tuscan validates "Flux can render bright detailed scenes." Cask 1 validates "atmospheric_horror prompts produce viable chiaroscuro." Cask 8 validates "deep catacomb darkness still has subjects." Cask 16 validates the extreme of darkness (the wall-sealing scene -- our hardest case). Cross-track validation rejected -- only atmospheric_horror has visual_prompts authored; other tracks would require Claude API spend to populate. Full 16-scene run rejected -- if scene 1 fails, the remaining 15 are wasted compute. |
| 5 | Vast.ai infrastructure resilience | **D: pre-flight script + tight rentals.** Combines (a) `scripts/preflight_check.py` runs HF token resolve / workflow validity / ComfyUI alive / score-server alive / workflow signature print before any GPU sampling; and (b) two tight rentals (~30 min Probe rental + ~45 min Cask validation rental), destroyed between major config changes. Pre-flight kills the "did the box even come up right" uncertainty per rental. Tight rentals limit hang exposure (S6.2 saw 2 host hangs in 3.5h). Total cost: ~$0.55, vs S6.2's ~$1.50 for one long rental. |
| 6 | Visual_prompts stage revision in scope? | **B: out of scope.** Don't change two variables at once. If the workflow rebuild + prefix drop produces detailed subjects on the existing prompts, prompt revision is unnecessary churn -- and risks regenerating all 16 scene prompts at additional Claude API cost (~$1 per re-adapt). The rebuild is the load-bearing change; prompt revision is contingent on Phase 2 outcomes. **Hard decision criterion** for whether revision is needed: see Section 9. |

### Explicit non-decision: `flux_pro_api` (still deferred)

PRD addition #5's BFL Pro API escape hatch remains the contingency if the
workflow rebuild fails to unlock viable atmospherics. We do not pursue it in
S6.3. Rationale unchanged from S6.2:

- Bypasses cloud-first GPU validation that S7-S13 depend on.
- Different code path (model routing) -- proper home is a per-track
  `image_model` flag with a dispatcher, not a one-off.
- We should actually try a **properly-configured** Flux Dev workflow before
  paying for hosted API. S6.2 disproved FP8 as the cause; S6.3's mission is
  proving (or disproving) the workflow-config hypothesis. Only if S6.3's

### 4b. `check_image_subject`

```python
def check_image_subject(
    path: Path,
    *,
    min_edge_density: float = 0.020,
) -> CheckResult:
    """Reject mood-only renders that lack subject boundaries.

    Brightness floor catches "fade to black" failures but not "abstract color
    field at high brightness" -- a yellow-and-dark color field can pass
    brightness floor (mean RGB ~100) and LAION (~4.3) yet contain no
    recognizable subjects. This primitive uses OpenCV Canny edge detection
    as a cheap proxy for "are there object boundaries in the image?".

    Edge density = count(Canny edges) / total pixels. Real photos and
    rendered scenes typically score 0.03-0.15. Smooth color fields score
    <0.01. Per-track threshold lives in track YAML's quality_gates block.
    """
    try:
        import cv2  # type: ignore[import-untyped]
        import numpy as np
        from PIL import Image
    except ImportError as exc:
        return CheckResult(
            passed=True,  # fail-open -- mirror check_hand_anomalies
            metric=0.0,
            threshold=min_edge_density,
            reason=f"skipped: {exc.name} unavailable",
        )
    try:
        with Image.open(path) as img:
            arr = np.asarray(img.convert("RGB"), dtype=np.uint8)
        gray = cv2.cvtColor(arr, cv2.COLOR_RGB2GRAY)
        edges = cv2.Canny(gray, 100, 200)
        density = float(np.count_nonzero(edges)) / float(edges.size)
    except Exception as exc:  # noqa: BLE001
        return CheckResult(
            passed=False,
            metric=0.0,
            threshold=min_edge_density,
            reason=f"failed to read image: {exc!r}",
        )
    if density >= min_edge_density:
        return CheckResult(
            passed=True,
            metric=density,
            threshold=min_edge_density,
            reason=f"ok (edge_density={density:.4f} >= min={min_edge_density:.4f})",
        )
    return CheckResult(
        passed=False,
        metric=density,
        threshold=min_edge_density,
        reason=f"no recognizable subject "
               f"(edge_density={density:.4f} < min={min_edge_density:.4f})",
    )
```

Pattern matches `check_image_brightness` exactly. Defensive ImportError
returns `passed=True` (fail-open) so missing cv2 doesn't halt the pipeline --
consistent with mediapipe drift fix. Failed PIL/cv2 read returns
`passed=False` because we DO want to reject unreadable files.

                  "save_image", "model_sampling_flux", "flux_guidance"}


def _check_hf_token(token: str) -> tuple[bool, str]:
    """GET 1KB range from gated FLUX.1-dev safetensors.

    Whoami can succeed while a token lacks gated download access; this is the
    real test. 200 -> token works; 401/403 -> token missing or unauthorized.
    """
    url = "https://huggingface.co/black-forest-labs/FLUX.1-dev/resolve/main/flux1-dev.safetensors"
    headers = {"Authorization": f"Bearer {token}", "Range": "bytes=0-1023"}
    try:
        with httpx.Client(follow_redirects=True, timeout=10) as client:
            resp = client.get(url, headers=headers)
        if resp.status_code in (200, 206):
            return True, f"HF token OK (got {resp.status_code} {len(resp.content)}B)"
        return False, f"HF token resolve failed: {resp.status_code} {resp.text[:200]}"
    except Exception as exc:  # noqa: BLE001
        return False, f"HF token resolve raised: {exc!r}"


def _check_workflow_json(path: Path) -> tuple[bool, str]:
    """Load JSON; verify _meta.role tags reference valid nodes."""
    try:
        data = json.loads(path.read_text())
    except Exception as exc:  # noqa: BLE001
        return False, f"workflow JSON load failed: {exc!r}"
    roles = data.get("_meta", {}).get("role", {})
    missing = REQUIRED_ROLES - set(roles)
    if missing:
        return False, f"workflow missing roles: {sorted(missing)}"
    for role, node_id in roles.items():
        if node_id not in data:
            return False, f"role '{role}' -> node '{node_id}' not in workflow"
    return True, f"workflow JSON OK ({len(data)} nodes, {len(roles)} roles)"


def _check_comfyui_alive(host: str) -> tuple[bool, str]:
    try:
        with httpx.Client(timeout=5) as client:
            resp = client.get(f"{host}/system_stats")
        if resp.status_code != 200:
            return False, f"ComfyUI {host}: {resp.status_code}"
        stats = resp.json()
        gpu_name = (stats.get("devices") or [{}])[0].get("name", "?")
        return True, f"ComfyUI alive at {host} (GPU={gpu_name})"
    except Exception as exc:  # noqa: BLE001
        return False, f"ComfyUI unreachable at {host}: {exc!r}"


def _check_score_server_alive(host: str) -> tuple[bool, str]:
    try:
        with httpx.Client(timeout=5) as client:
            resp = client.get(f"{host}/health")
        if resp.status_code != 200:
            return False, f"score-server {host}: {resp.status_code}"
        return True, f"score-server alive at {host}"
    except Exception as exc:  # noqa: BLE001
        return False, f"score-server unreachable at {host}: {exc!r}"


def _workflow_signature(path: Path) -> str:
    """sha256 of canonical-form JSON (sorted keys)."""
    data = json.loads(path.read_text())
    canonical = json.dumps(data, sort_keys=True, separators=(",", ":")).encode()
    return hashlib.sha256(canonical).hexdigest()[:12]

### 4e. `scripts/reset_scene_keyframes.py`

```python
"""Reset keyframe state for specified scenes in a story.

Needed because the orchestrator skips scenes whose keyframe_path is set.
After the S6.2 Phase 2 dirty state, scenes 1/8/16 carry stale keyframe_path
values pointing at black/abstract PNGs from the disproven hypothesis. To
re-run those scenes against the rebuilt workflow, we either need to clear
the path or append a failed StageRun -- this script is the cleaner option.

Usage:
    python scripts/reset_scene_keyframes.py STORY_ID --scenes "1,8,16"
                                            [--delete-files] [--story-dir PATH]
"""
import argparse
import json
import sys
from pathlib import Path


def reset_scenes(
    *, story_id: str, scenes: list[int], story_dir: Path, delete_files: bool
) -> int:
    story_path = story_dir / story_id / "story.json"
    if not story_path.exists():
        print(f"ERROR: {story_path} not found", file=sys.stderr); return 2
    data = json.loads(story_path.read_text())
    keyframes_dir = story_dir / story_id / "keyframes"
    reset_count = 0
    for scene in data.get("scenes", []):
        if scene.get("index") not in scenes:
            continue
        scene["keyframe_path"] = None
        scene["keyframe_candidates"] = []
        scene["keyframe_scores"] = []
        validation = scene.get("validation", {})
        validation.pop("keyframe_anatomy", None)
        validation.pop("keyframe_selected_via_fallback", None)
        scene["validation"] = validation
        reset_count += 1
        if delete_files:
            scene_subdir = keyframes_dir / f"scene_{scene['index']:03d}"
            if scene_subdir.exists():
                for png in scene_subdir.glob("candidate_*.png"):
                    png.unlink()
    # Atomic write.
    tmp = story_path.with_suffix(".json.tmp")
    tmp.write_text(json.dumps(data, indent=2))
    tmp.replace(story_path)
    print(f"reset {reset_count} scenes in {story_path}", flush=True)
    return 0


def main() -> int:
    parser = argparse.ArgumentParser()
    parser.add_argument("story_id")
    parser.add_argument("--scenes", required=True,
                        help="comma-separated scene indices (e.g. 1,8,16)")
    parser.add_argument("--delete-files", action="store_true",
                        help="also delete candidate_*.png from disk")
    parser.add_argument("--story-dir", type=Path, default=Path("data/stories"))
    args = parser.parse_args()

    scenes = sorted({int(s.strip()) for s in args.scenes.split(",") if s.strip()})
    return reset_scenes(
        story_id=args.story_id,
        scenes=scenes,
        story_dir=args.story_dir,
        delete_files=args.delete_files,
    )


if __name__ == "__main__":
    sys.exit(main())
```

## 6. Error handling

| Failure mode | Behaviour |
|---|---|
| Single candidate fails brightness floor | Same as S6.2: skip subject + LAION + anatomy, scoring_succeeded=False, log WARN. |
| Single candidate fails subject gate | Same shape as brightness-fail: skip LAION + anatomy, scoring_succeeded=False, log WARN with edge_density value. |
| All 3 candidates fail brightness | `KeyframeGenerationError` -- existing S6.2 halt rule. |
| All 3 candidates fail subject gate | `KeyframeGenerationError` -- NEW halt rule, mirrors brightness halt. |
| Brightness + subject + scoring partial mix | Eligibility filter ANDs all three. Fallback uses (`scoring_succeeded AND brightness_passed AND subject_passed`) set. Subject-failing candidates NEVER picked, even in fallback. |
| Track YAML missing `subject_min_edge_density` | Default 0.020 (matches atmospheric_horror; permissive). Logged INFO once per stage run. |
| `check_image_subject` ImportError on cv2 or numpy | Returns `passed=True, reason="skipped: cv2 unavailable"`. Pipeline continues -- fail-open mirrors mediapipe defensive handler. |
| `check_image_subject` PIL/cv2 decode error | Returns `passed=False`. Treated same as gate failure. |
| Pre-flight HF token resolve fails | Exit nonzero from preflight; runbook fails-fast before any GPU spend. |
| Pre-flight workflow JSON invalid | Same -- exit nonzero; signals workflow rebuild has a bug to fix locally before box is wasted. |
| Pre-flight ComfyUI/score-server unreachable | Same -- the launchers in vast_setup.sh failed; investigate before any sampling. |
| `reset_scene_keyframes.py` story.json not found | Exit nonzero with clear stderr. Idempotent on re-run after fixing path. |
| Pre-existing failure modes (ComfyUI crash mid-run, score_server OOM, vast.ai network hang) | All preserved from S6.1/S6.2; subject gate is purely additive. |

def make_high_edge_png(path: Path, *, width: int = 512, height: int = 512) -> Path:
    """Deterministic checkerboard. Edge density >0.10."""
    import numpy as np
    from PIL import Image
    cell = 16
    arr = np.zeros((height, width, 3), dtype=np.uint8)
    for y in range(height):
        for x in range(width):
            if ((x // cell) + (y // cell)) % 2 == 0:
                arr[y, x] = (255, 255, 255)
    Image.fromarray(arr, "RGB").save(path)
    return path


def make_low_edge_png(path: Path, *, width: int = 512, height: int = 512) -> Path:
    """Smooth horizontal gradient. Edge density <0.005."""
    import numpy as np
    from PIL import Image
    arr = np.zeros((height, width, 3), dtype=np.uint8)
    for x in range(width):
        v = int(255 * x / width)
        arr[:, x] = (v, v, v)
    Image.fromarray(arr, "RGB").save(path)
    return path


def make_solid_color_png(
    path: Path, *, color: tuple[int, int, int] = (127, 127, 127),
    width: int = 64, height: int = 64,
) -> Path:
    """Solid color fill. Edge density = 0."""
    from PIL import Image
    Image.new("RGB", (width, height), color).save(path)
    return path

### Three intentional half-measures worth flagging

1. **Subject gate is edge-density only.** A high-noise random-noise image
   would also have high edge density and pass the gate. We have not
   observed this failure mode on live data; if S7's full run produces a
   "noise pattern" failure, we add an entropy primitive then.

2. **Probe sequence is sequential, not parallel.** We could run Probe A
   and Probe B against two simultaneous A6000 boxes for faster decision
   time (~10 min instead of ~30). Parallel rentals double the host-hang
   exposure window AND require either two SSH tunnels or two VPN endpoints
   to be live. Sequential keeps it simple.

3. **`subject_min_edge_density` is a single per-track threshold, not a
   per-scene-darkness-class threshold.** The wall-sealing scene 16 is
   intrinsically darker than scene 1; a single threshold may be too tight
   for 16 and too loose for 1. If the calibration step in 8b shows
   inter-scene variance is wide, the threshold may need to be relaxed AND
   the eye-check made the primary gate for atmospheric_horror specifically.

## 10. Lessons carried in (and one that may be reinforced)

1. **Late binding for testability.** `check_image_subject` is a pure
   function; no factory. Same as S5's `check_*` primitives.

2. **Recorder/Fake protocol pattern.** No new Protocols in S6.3.

3. **Pure-core / impure-shell at the Stage boundary.** Subject gate lives
   in `generate_for_scene` (pure async). Stage shell unchanged from
   S6.1/S6.2.

4. **MockTransport / FastAPI TestClient seam unchanged.** S6.3's
   `preflight_check.py` HTTP tests reuse the MockTransport pattern.

5. **Per-stage atomic save preserved.** Subject halt path uses the same
   best-effort `story.save(...)` before re-raise as S6.1's scoring halt.

6. **Subagent scope drift on chore tasks.** S6.2 incidents (Task 6 batch
   drift, Task 13 single-flag drift) reinforced this. Implementer prompts
   for S6.3 will explicitly forbid: dropping any line/flag/value not in
   the explicit drop list (even if "obviously redundant"); adding
   Co-Authored-By trailers; making "small fixes" to unrelated files;
   committing `data/stories/*` (regenerated artifacts will be ~150-300KB
   x 9 + ~150-300KB x 6 = ~2-4MB easy to accidentally commit).

7. **Final cumulative review catches gaps individual reviews miss.** Same
   discipline as S6.1/S6.2.

8. **Perceptual safety nets need cheap pre-filters.** S6.2's lesson now
   has TWO instances: brightness floor (S6.2) and subject gate (S6.3).
   The pattern is firmly established: every score-based gate gets a
   cheap pre-filter that rejects out-of-distribution inputs before they
   pollute the score. Generalizes to S8 (motion-flow on stills), S9
   (PSNR on failed upscale), S13 (LUFS on silent track).

### One lesson S6.3 will likely produce

9. **NEW (provisional): Always run a known-good config before designing
   a fix.** S6.2 Phase 1 spent 12 task-commits + ~$1.50 of cloud rental
   designing a fix for FP8 collapse, but the actual root cause was a
   missing-node workflow misconfiguration that ComfyUI's reference Flux
   workflow would have caught in 30 minutes. The deeper learning: when an
   offline hypothesis assumes a cause of perceptual collapse, ALWAYS
   benchmark against the upstream reference config before designing a
   custom fix. Pre-flight script (4d) institutionalizes this -- the
   workflow signature print acts as a "did we drift from the reference?"
   tripwire on every rental.
